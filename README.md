# 重返未来1999 - 抽卡统计

一个用于记录和统计《重返未来1999》抽卡数据的项目，包含抽卡记录和角色信息，提供网页可视化展示。

## 项目结构

```
Reverse-1999/
├── characters/          # 角色数据目录
│   ├── 2/gainIds.josn   # 2星角色列表
│   ├── 3/gainIds.josn   # 3星角色列表
│   ├── 4/gainIds.josn   # 4星角色列表
│   ├── 5/gainIds.josn   # 5星角色列表
│   └── 6/gainIds.josn   # 6星角色列表
├── poolType/            # 卡池类型目录
│   ├── 1/               # 新人 & 常驻征集
│   ├── 3/               # UP池（活动池）
│   ├── 6/               # 限定征集
│   ├── 7/               # 限时活动征集
│   └── 12/              # 湖的涟漪征集
├── pageData.josn        # 抽卡记录数据
├── index.html           # 抽卡统计网页
├── icon.png             # 网页头像图标
└── README.md            # 项目说明
```

## 数据文件说明

### characters/ 目录

存放各稀有度角色的ID和名称映射，格式为：

```
角色ID:角色名称,
```

例如：

```
3146:小瑞安侬,
3154:银装小春雀儿,
```

### poolType/ 目录

存放各卡池类型的抽卡记录，目录名对应 `poolType` 字段值，文件名即为卡池名称。每个文件第一行可能包含 UP 角色信息：

```
up:角色ID:角色名称,
{"summonType":"10","luckyBagIds":[],"createTime":"2026-05-21 12:25:16","poolId":37121,"gainIds":[...],"poolType":3,"poolName":"剥茧为名"},
```

### pageData.josn

存放抽卡记录，每条记录包含：

| 字段 | 类型 | 说明 |
|------|------|------|
| summonType | string | 抽卡类型，`"1"` 为单抽，`"10"` 为十连 |
| poolName | string | 卡池名称 |
| poolType | number | 卡池类型ID（见下方说明） |
| poolId | number | 卡池ID |
| gainIds | array | 本次抽卡获得的角色ID列表 |
| createTime | string | 抽卡时间，格式 `YYYY-MM-DD HH:MM:SS` |
| luckyBagIds | array | 幸运袋获得的角色ID列表（通常为空） |
| draws | number | 获得6星角色时的抽数（编辑模式下可手动修改） |
| isUp | boolean | 是否为UP角色（编辑模式下可手动设置） |

## 名词解释

### 稀有度等级

游戏中角色稀有度分为5个等级：

- **6星**：最高稀有度角色，强度通常最高
- **5星**：高级稀有度角色
- **4星**：中级稀有度角色
- **3星**：普通稀有度角色
- **2星**：最低稀有度角色

### 卡池类型 (poolType)

| poolType | 类型 | 说明 |
|----------|------|------|
| 1 | 新人 & 常驻征集 | 普通征集，包含常驻角色，抽数按池子独立计算 |
| 3 | UP池（活动池） | 限定UP角色概率提升，**抽数累计跨池子继承**，活动结束清零 |
| 6 | 限定征集 | 限定角色专属卡池，抽数按池子独立计算 |
| 7 | 限时活动征集 | 限时活动卡池，如湖的邀约、湖的许诺、湖的应许等，每个池子独立保底，活动结束清零 |
| 12 | 湖的涟漪征集 | 涟漪系列活动卡池，保底次数与同类征集共享，**活动结束不清零** |

### 十连

一次性抽取10次，通常有保底机制（第10次必定获得3星及以上角色）。

### 歪卡

抽到非UP角色的6星角色。例如，在"故事的新啼"卡池中抽到小瑞安侬，但该卡池的UP角色并不是小瑞安侬，即为歪卡。

### 保底机制

在征集活动中，若连续60次征集未获取6星角色，则下次征集获取6星角色的概率将从1.5%提升至4%。若该次征集仍未获取6星角色，则后续每次征集获取6星角色的概率将继续依照2.5%累加递升。最多70次征集必定能通过保底获取6星角色。

- **UP池（poolType=3）**：保底次数跨池子继承，活动结束清零
- **湖的涟漪（poolType=12）**：保底次数与同类征集共享，活动结束不清零
- **其他池子**：每个池子独立计算保底

## 抽卡统计网页

打开 `index.html` 即可查看抽卡统计，网页会自动读取数据文件并展示：

- **统计数据**：六星获取数、歪卡次数、累计抽数、UP池已垫抽数、湖涟漪已垫抽数
- **保底说明**：显示当前UP池和湖涟漪已垫抽数、距概率提升和保底的剩余抽数
- **卡池列表**：每个卡池的抽数和获得的六星角色（从新到旧排序）
- **抽数计算**：按时间顺序精确计算获得六星角色所花费的抽数
- **隐藏空池**：滑动开关，可隐藏暂无六星的卡池
- **更新数据**：通过弹窗输入征集分析链接，自动获取并合并抽卡数据
- **编辑模式**：进入编辑模式后可修改抽卡记录
- **新增池子**：在编辑模式下可新建卡池（支持填写累计抽数）
- **设置**：配置 GitHub Personal Access Token

### 编辑模式功能

进入编辑模式后，可以：

1. **修改出货记录**：点击六星角色卡片，可修改角色名称、星级、抽数和是否UP状态
2. **新增出货**：在每个池子的六星列表末尾点击"+新增"按钮，可添加新的出货记录
3. **新建池子**：点击页面底部"+新增池子"按钮，可创建新卡池
4. **保存修改**：退出编辑模式时可选择保存修改，数据会自动同步到 GitHub

## 运行方式

1. 使用浏览器直接打开 `index.html`（需要本地服务器支持）
2. 或启动本地服务器：

```bash
python -m http.server 8001
```

然后访问 `http://localhost:8001/index.html`

## 更新数据

### 方式一：网页更新

1. 点击网页右下角"更新"按钮
2. 输入征集分析链接（格式：`https://game-re-service.sl916.com/*`）
3. 点击"获取并合并数据"，数据将自动去重并合并

### 方式二：手动更新

直接编辑 `pageData.josn` 文件，添加新的抽卡记录，刷新网页即可。

### 方式三：编辑模式

1. 点击右下角"修改"按钮进入编辑模式
2. 在各池子中修改或添加出货记录
3. 再次点击"修改"按钮退出并保存修改
4. 数据会自动推送到 GitHub 仓库

## GitHub 同步

配置 GitHub Personal Access Token 后，修改的数据会自动推送到仓库：

1. 点击右下角"设置"按钮
2. 输入 GitHub Personal Access Token（需要有 repo 权限）
3. 点击"保存"按钮
4. 之后在编辑模式下修改数据并保存时，会自动推送到 GitHub
